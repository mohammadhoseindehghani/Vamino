# 🏦 وامینو (Vamino) | سیستم مدیریت هوشمند وام و ضامن

**وامینو** یک پلتفرم پیشرفته و مقیاس‌پذیر برای مدیریت فرآیندهای مالی است که با تمرکز بر شفافیت در ثبت قراردادهای وام و مدیریت سیستمی ضامن‌ها طراحی شده است. این پروژه با استفاده از جدیدترین ابزارهای اکوسیستم دات‌نت، تجربه‌ای روان و ایمن برای کاربران فراهم می‌کند.

---

## 🚀 چرا وامینو؟
در سیستم‌های سنتی، فرآیند تأیید ضامن و پیگیری وضعیت وام معمولاً با کندی و سردرگمی همراه است. وامینو با خودکارسازی این فرآیندها، نقش پل ارتباطی میان «وام‌گیرنده» و «ضامن» را ایفا می‌کند و با منطقِ اعتبار‌سنجی هوشمند، مدیریت این روابط را ساده کرده است.

---

## 🛠 تکنولوژی‌های مورد استفاده

این پروژه ویترینی از ابزارهای مدرن .NET است:

*   **Framework:** .NET 10 (ASP.NET Core MVC)
*   **Architecture:** Clean Architecture + CQRS (MediatR)
*   **Authentication:** ASP.NET Core Identity
*   **Data Access:** Entity Framework Core
*   **Validation:** FluentValidation
*   **Logging:** Serilog
*   **UI/UX:** Responsive Design (Bootstrap/CSS)

---

## 💡 ویژگی‌های کلیدی سیستم

### ۱. مدیریت هوشمند قراردادهای وام
*   **محاسبه خودکار ضامن:** سیستم به ازای هر ۵۰ میلیون تومان، ضامن جدیدی درخواست می‌کند (قانون خودکار برای جلوگیری از خطای انسانی).
*   **وضعیت‌دهی پویا:** قراردادها در وضعیت‌های مختلف (در انتظار، تکمیل شده، رد شده) به‌صورت بلادرنگ مدیریت می‌شوند.
*   **کنترل دسترسی:** کاربر نمی‌تواند در یک قرارداد واحد، هم‌زمان نقش «وام‌گیرنده» و «ضامن» را داشته باشد.

### ۲. پنل کاربری جامع
*   **برای وام‌گیرنده:** ثبت درخواست، ویرایش یا لغو وام (تا قبل از نهایی شدن قرارداد)، و پیگیری لحظه‌ای وضعیت ضامن‌ها.
*   **برای ضامن:** مشاهده درخواست‌های ضمانت، امکان پذیرش یا رد درخواست همراه با ثبت یادداشت (Comment) برای شفافیت بیشتر.

### ۳. امنیت و پایداری
*   **Error Handling:** استفاده از Middleware اختصاصی برای کنترل خطاهای سیستم و جلوگیری از نمایش خطاهای خام به کاربر.
*   **تزریق وابستگی (Dependency Injection):** استفاده حداکثری از اصول SOLID برای افزایش تست‌پذیری کد.

---

## 🏗 ساختار معماری (Clean Architecture)

این پروژه با تفکیک وظایف زیر پیاده‌سازی شده است:

1.  **Domain:** حاوی موجودیت‌ها و قوانین تجاری (Business Rules) بدون وابستگی به خارج.
2.  **Application:** مدیریت UseCaseها و جدا‌سازی فرامین (Commands) و پرس‌وجوها (Queries) با MediatR.
3.  **Infrastructure:** پیاده‌سازی لایه دسترسی به داده (EF Core) و تنظیمات لاگ‌گذاری.
4.  **Presentation:** لایه MVC برای ارتباط مستقیم با کاربر.
